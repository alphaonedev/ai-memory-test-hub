## Day 1 — Gates

### Ship-gate Phase 1 — Functional (~30 min)

Run on the `release/v0.6.3` build. Single-node, exercises every MCP tool + every HTTP route.

```
cd ai-memory-ship-gate
./scripts/phase1_functional.sh release/v0.6.3
./scripts/collect_reports.sh phase1 v0.6.3
```

**Acceptance:** all MCP tools return expected shapes; capabilities reports `schema_version=2`; HTTP 4xx on bad input.

### Ship-gate Phase 2 — Multi-agent (~1 hour)

Federation W=2 of N=3 quorum. Sustained 100-row burst from node-1.

```
./scripts/phase2_multiagent.sh release/v0.6.3
```

**Acceptance:** every row reachable from every peer post-burst. mTLS fingerprint allowlist refuses unallowlisted peer. namespace_meta propagation correct on all 3 nodes (S35).

### Ship-gate Phase 3 — Migration (~30 min)

v0.6.2 → v0.6.3 schema migration on a populated DB.

```
./scripts/phase3_migration.sh v0.6.2 v0.6.3
```

**Acceptance:** v15 schema applied (memory_links gains 4 nullable columns + 3 temporal indexes; entity_aliases table created). Existing rows queryable. Backfill correct (`valid_from = source.created_at`).

### Ship-gate Phase 4 — Chaos (abbreviated, ~2 hours)

```
./scripts/phase4_chaos.sh release/v0.6.3 --abbreviated
```

Abbreviated = single chaos cycle each: peer kill mid-burst (1 cycle), network partition (1 cycle), clock skew (1 cycle). Soak windows (3+ days) deferred.

**Acceptance:** zero data loss after each cycle. Pending actions converge. No daemon crashes.

## Day 2 — Tag + release

### Tag v0.6.3 (~30 min)

```
git checkout release/v0.6.3
git tag -s v0.6.3 -m "v0.6.3 — Hierarchy, KG, Capabilities v2, 93.05% coverage"
git push origin v0.6.3
```

Signed tag fires the release pipeline.

### Watch release pipeline (~2h wall, mostly idle)

5 platform binaries built + signed + SBOM attached. Subsequent jobs: GitHub Release, crates.io publish, Docker GHCR, Homebrew, Ubuntu PPA, Fedora COPR.

**Acceptance:** every job COMPLETED green; no SKIPPED that should have run.

### Distribution-channel post-publish smoke (~2h, parallel)

For each channel, install + verify version + start daemon + check capabilities returns `schema_version=2`:

- `brew install alphaonedev/tap/ai-memory`
- `cargo install ai-memory --version 0.6.3`
- `docker pull ghcr.io/alphaonedev/ai-memory:0.6.3`
- `apt install ai-memory` (Ubuntu PPA)
- `dnf install ai-memory` (Fedora COPR)

### Sync release/v0.6.3 → main (~30 min)

After publish verified:
```
git checkout main
git merge --no-ff release/v0.6.3
git push origin main
```

This brings the v0.6.3 code (capabilities v2, coverage gate) into main alongside the docs that landed via PRs #458/#460/#461.
